Pendahuluan

GitHub Pages adalah layanan hosting web statis yang memungkinkan Anda menerbitkan situs web langsung dari repositori GitHub Anda. Ini ideal untuk membuat situs web pribadi, portofolio, atau dokumentasi proyek. Tutorial ini akan memandu Anda langkah demi langkah dalam membuat dan menerbitkan situs web menggunakan GitHub Pages.

Prasyarat

Sebelum memulai, pastikan Anda memenuhi persyaratan berikut:

  • Akun GitHub. Jika Anda belum memilikinya, daftar di GitHub.
  • Pengetahuan dasar tentang HTML dan CSS. Ini akan membantu Anda dalam mengedit dan menyesuaikan situs web Anda.

Langkah-Langkah Membuat Website di GitHub Pages

1. Membuat Repositori GitHub

  1. Masuk ke akun GitHub Anda.
  2. Klik tombol "New" di sebelah kiri atas untuk membuat repositori baru.
  3. Beri nama repositori Anda. Nama repositori ini akan menentukan URL situs web Anda. Misalnya, jika nama repositori Anda adalah mywebsite, URL situs web Anda akan menjadi https://username.github.io/mywebsite/.
  4. Pilih opsi "Public" agar repositori dapat diakses secara publik.
  5. Centang "Initialize this repository with a README" untuk membuat file README.
  6. Klik "Create repository".

2. Menambahkan Berkas Website

  1. Di halaman repositori Anda, klik "Add file" dan pilih "Upload files".
  2. Unggah berkas HTML, CSS, dan gambar Anda. Jika Anda baru mulai, Anda bisa membuat berkas HTML dan CSS sederhana.
  3. Klik "Commit changes" untuk menyimpan perubahan Anda.

3. Mengaktifkan GitHub Pages

  1. Di halaman repositori Anda, klik "Settings".
  2. Gulir ke bawah ke bagian "Pages".
  3. Di bawah "Source", pilih "main" atau "master" dari menu dropdown.
  4. Klik "Save".

GitHub Pages akan mulai memproses situs web Anda dan akan memberi Anda URL untuk mengaksesnya. Biasanya, proses ini memerlukan beberapa menit.

4. Mengedit dan Menyesuaikan Situs Web

  1. Kembali ke repositori Anda dan buka berkas HTML atau CSS yang ingin Anda edit.
  2. Klik "Edit" untuk memodifikasi berkas tersebut secara langsung di GitHub.
  3. Setelah selesai mengedit, klik "Commit changes" untuk menyimpan perubahan.

5. Menambahkan Fitur dan Konten

  • Menambahkan Halaman Baru: Buat berkas HTML baru untuk menambahkan halaman seperti "Tentang Kami" atau "Kontak".
  • Menggunakan Tema: Anda bisa menggunakan tema Jekyll untuk menambahkan desain yang lebih menarik. Jekyll adalah generator situs statis yang terintegrasi dengan GitHub Pages.
  • Menambahkan Gambar dan Media: Unggah gambar dan file media ke repositori dan referensikan dalam berkas HTML Anda.

Tips dan Trik

  • Gunakan Jekyll: Untuk pengalaman yang lebih kaya dan kontrol lebih banyak, pertimbangkan untuk menggunakan Jekyll. GitHub Pages mendukung Jekyll secara langsung, dan Anda bisa memulai dengan dokumentasi Jekyll.
  • Pemeriksaan Browser: Selalu periksa situs web Anda di berbagai browser dan perangkat untuk memastikan kompatibilitas.
  • Kustomisasi Lebih Lanjut: Eksplorasi CSS dan JavaScript untuk menambahkan interaktivitas dan gaya kustom pada situs web Anda.
